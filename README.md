# 快閃卡 FlashCard 使用者操作手冊 📚

快閃卡 FlashCard 是一套桌面版記憶卡工具，適合用來準備考試、背單字、整理問答題、記憶專業知識或建立個人題庫。你可以建立多個牌組、編輯正反面卡片、插入圖片、用複習模式安排記憶，也可以用測驗模式檢查自己是否真的記住。

## 快速開始 🚀

如果你已經拿到打包好的 Windows 版本：

1. 開啟 `portable/FlashCard/FlashCard.exe`。
2. 在左側點選「+ 新增牌組」。
3. 輸入牌組名稱，例如「英文單字」、「期末考重點」。
4. 點選牌組後，按「+ 新增卡片」。
5. 填寫正面題目與反面答案，按「儲存」。
6. 回到牌組後，可使用「複習模式」或「測驗模式」開始練習。

## 主畫面介紹 🖥️

畫面主要分成兩個區域：

- 左側側邊欄：顯示所有牌組，可新增牌組、切換牌組、刪除牌組，也可以進入「匯入 / 匯出」。
- 右側主要區域：依目前選擇顯示牌組列表、卡片列表、複習模式、測驗模式或匯入匯出工具。

剛開啟程式時，如果還沒有任何資料，畫面會提示你先新增牌組或匯入既有資料。

## 建立與管理牌組 🗂️

### 新增牌組 ➕

1. 在左下角點選「+ 新增牌組」。
2. 在「牌組名稱」輸入名稱。
3. 可在「描述」輸入補充說明，這是選填欄位。
4. 點選「新增」完成。

### 開啟牌組 📖

在左側側邊欄或主畫面的牌組卡片上點選牌組，即可查看該牌組內的所有卡片。

### 刪除牌組 🗑️

將滑鼠移到牌組名稱或牌組卡片上，點選垃圾桶或「刪除」可刪除牌組。系統會先跳出確認訊息，確認後才會真正刪除。

注意：刪除牌組會同時移除該牌組內的卡片。操作前建議先匯出 JSON 備份。

## 新增與編輯卡片 ✏️

### 新增卡片

1. 先進入一個牌組。
2. 點選右上角「+ 新增卡片」。
3. 在「正面（題目）」輸入題目。
4. 在「反面（答案）」輸入答案。
5. 確認右側預覽內容正確。
6. 點選「儲存」。

「正面」與「反面」都必須有內容，才能儲存卡片。

### 編輯卡片

1. 進入牌組後，在卡片列表找到要修改的卡片。
2. 將滑鼠移到卡片上。
3. 點選「編輯」。
4. 修改正面或反面內容。
5. 點選「儲存」。

### 刪除卡片

1. 進入牌組後，將滑鼠移到卡片上。
2. 點選「刪除」。
3. 卡片會從目前牌組移除。

## 使用 Markdown 排版 📝

卡片內容支援 Markdown 與常見 GFM 語法。你可以用簡單符號整理內容：

```markdown
# 大標題
## 小標題

**粗體重點**
*斜體補充*

- 條列項目
- 第二點

`程式碼或關鍵字`
```

建議讓「正面」保持簡短明確，將完整解釋放在「反面」，複習時會比較有效率。

## 插入圖片 🖼️

卡片正面與反面都可以加入圖片，適合放公式、圖表、截圖、流程圖或手寫筆記。

### 用按鈕插入圖片

1. 開啟新增或編輯卡片視窗。
2. 在正面或反面區塊點選「插入圖片」。
3. 選擇圖片檔案。
4. 程式會自動把圖片加入文字區，並在右側預覽顯示。

支援格式包含：PNG、JPG、JPEG、GIF、WEBP、BMP、SVG。

### 貼上圖片

1. 複製一張圖片，例如截圖或剪貼簿圖片。
2. 點進正面或反面的文字區。
3. 直接貼上。
4. 圖片會自動存入卡片資料並加入 Markdown 內容。

### 拖放圖片

1. 從檔案總管拖曳圖片。
2. 放到正面或反面的文字區。
3. 圖片會自動插入到卡片內容中。

如果看到「無法讀取選取的圖片」，請確認檔案仍存在、格式受支援，並重新選擇圖片。

## 預覽與翻卡 👀

新增或編輯卡片時，右側會即時顯示正面與反面的預覽。

在卡片列表中點選任一卡片，會進入單張卡片預覽：

- 一開始顯示正面。
- 點擊卡片可翻到反面。
- 點選「回到列表」可返回牌組。

## 複習模式 🔁

複習模式會依照卡片的複習狀態安排要複習的內容，包含到期卡片與新卡片。

### 開始複習

1. 進入一個牌組。
2. 點選「複習模式」。
3. 先看正面題目。
4. 點擊卡片或按空白鍵查看答案。
5. 依照自己的記憶狀況評分。

### 評分選項

- 重來 `[1]`：完全忘記，應該很快再看一次。
- 困難 `[2]`：想起來了，但很不熟。
- 良好 `[3]`：正常記得。
- 簡單 `[4]`：非常熟，可以隔久一點再複習。

評分後，系統會更新卡片的複習間隔與下次複習日期。

### 快捷鍵 ⌨️

- 空白鍵：翻面或回到正面。
- `1`：重來。
- `2`：困難。
- `3`：良好。
- `4`：簡單。

評分快捷鍵需要先翻到答案面才會生效。

## 測驗模式 🧪

測驗模式會從目前牌組隨機抽取最多 20 張卡片，讓你輸入答案並即時核對。

### 開始測驗

1. 進入一個牌組。
2. 點選「測驗模式」。
3. 閱讀題目。
4. 在輸入框輸入答案。
5. 點選「提交答案」，或按 Enter。
6. 看完結果後點選「下一題」，或再按 Enter。

### 判定方式

系統會將你的答案與卡片反面文字比對。比對時會忽略前後空白與英文大小寫，但不會理解同義詞或不同寫法。

如果答案很長、包含 Markdown、圖片或多段說明，建議用複習模式為主，測驗模式則適合短答案、單字、年份、名詞解釋等內容。

### 測驗結果 🎯

完成後會顯示：

- 答對題數
- 答錯題數
- 正確率

你可以選擇「再測一次」重新抽題，或「回到牌組」繼續管理卡片。

## 匯入與匯出 💾

點選左側「匯入 / 匯出」，可備份資料或從外部檔案建立卡片。

### 匯出 JSON

JSON 是完整備份格式，會保留：

- 牌組
- 卡片
- 複習進度
- 間隔重複資料

建議定期使用「匯出為 JSON（完整備份，含 SRS 進度）」保存備份。

### 從 JSON 匯入

1. 點選「從 JSON 匯入（完整備份還原）」。
2. 選擇先前匯出的 JSON 檔。
3. 匯入後，資料會合併到目前資料中。

### 匯出 CSV

CSV 適合用 Excel、Google Sheets 或其他試算表工具整理題庫。

匯出的欄位為：

```csv
deckName,front,back,starRating
```

CSV 包含牌組名稱、正面、反面與手動星等，不會保存 SRS 複習間隔、下次複習日期等完整學習進度。

### 從 CSV 匯入

CSV 檔案至少需要 `front` 與 `back` 欄位，建議格式如下：

```csv
deckName,front,back,starRating
英文單字,apple,蘋果,3
英文單字,book,書,0
```

`deckName` 可省略。若省略，系統會使用 `Default` 作為牌組名稱。`starRating` 可省略，省略時預設為 0。

## 資料保存與備份 🛡️

程式會把資料存在本機應用程式資料目錄中。一般使用時不需要手動管理這些檔案。

建議的備份方式：

1. 每次大量新增或修改卡片後，匯出 JSON。
2. 將 JSON 備份放到雲端硬碟或外接硬碟。
3. 若要搬到另一台電腦，先在原電腦匯出 JSON，再到新電腦匯入。

如果你只想用 Excel 整理題目，可匯出 CSV；如果要完整保留學習進度，請使用 JSON。

## 常見問題 ❓

### 為什麼「儲存」按鈕不能按？

正面與反面都必須輸入內容。請確認題目與答案不是空白。

### 為什麼複習模式沒有卡片？

可能原因：

- 目前牌組沒有卡片。
- 所有已學卡片還沒到下次複習時間。

你可以新增新卡片，或改用測驗模式練習目前牌組。

### 為什麼測驗答案明明意思相同卻被判錯？

測驗模式採文字比對，不會判斷語意。請讓答案盡量短而固定，例如單字、日期、公式結果或專有名詞。

### 圖片插入後會存在哪裡？

圖片會複製到程式的本機資料資料夾，卡片中會以 `images/...` 的方式引用。一般使用者不需要手動移動圖片。

### 匯入 JSON 或 CSV 會覆蓋原資料嗎？

目前匯入會將資料合併到現有資料中，不會先清空原資料。若匯入相同內容，系統會略過重複卡片，並在匯入結果中顯示新增與略過的數量。

### 刪除牌組或卡片可以復原嗎？

程式內沒有復原功能。刪除牌組前會要求再次確認；刪除卡片則會立即移除。大量整理前建議先匯出 JSON 備份。

## 建議使用流程 ✅

1. 先依科目或主題建立牌組。
2. 每張卡片只放一個重點。
3. 正面寫問題，反面寫答案與解釋。
4. 需要圖片時直接貼上或拖放。
5. 每天用複習模式處理到期卡片。
6. 考前用測驗模式快速檢查熟悉度。
7. 定期匯出 JSON 備份。

## 進階：開發與建置 🛠️

以下內容提供給需要從原始碼啟動或打包的人。

### 系統需求

- Node.js LTS
- npm
- Rust toolchain
- Windows 上建置 Tauri 需要 Microsoft C++ Build Tools

主要前端技術包含 React 19、TypeScript、Vite、Tailwind CSS、Tauri 2、`react-markdown`、`remark-gfm` 與 `lucide-react`。

### 安裝依賴

```bash
npm install
```

### 啟動開發模式

```bash
npm run tauri:dev
```

Windows 也可以執行：

```bat
dev.bat
```

### 建置前端

```bash
npm run build
```

### 打包桌面程式

```bash
npm run tauri:build
```

Windows portable EXE 包裝流程：

```bat
build.bat
```

輸出位置：

```text
portable/FlashCard/FlashCard.exe
```

### 品質檢查

```bash
npm run lint
npm test
npm run build
```

調整桌面端功能時，也建議執行：

```bash
cd src-tauri
cargo check
```
